# Branches Reference Examples

Reference tables of specific Branches with statistical mechanics, representing significant adult experiences, training, and career paths. These examples show how life events can mechanically enhance characters.

## Branch Mechanics Overview

Branches represent adult life experiences and provide:
- **Age Addition:** Years spent in this experience/training
- **Karma Cost (KV):** GM karma investment required
- **Prerequisites:** Requirements to access this Branch
- **Skills:** Professional or experiential training gained
- **Attributes:** Development from experience
- **Equipment/Resources:** Tools, connections, or assets gained
- **Special Abilities:** Unique capabilities from the experience

## GM Implementation Guidelines

### Prerequisites and Logic
- Prerequisites ensure Branches logically follow from character background
- Age additions reflect realistic time investment for expertise development
- Skills gained should match the professional/experiential focus
- Attributes improve in ways that make sense for the experience

### Karma Investment Strategy
- Higher KV Branches provide more significant benefits
- GMs balance powerful abilities with appropriate karma costs
- Multiple Branches possible but increase total age and karma investment
- Branch combinations should create coherent character narratives

### Campaign Integration
- Branch connections provide ongoing adventure hooks
- Professional networks create information and resource access
- Special abilities offer unique problem-solving options
- Resources and equipment support character role in party

### Balance Considerations
- No Branch should overshadow character's core capabilities
- Benefits should enhance rather than replace base character abilities
- Age accumulation affects character lifespan and development timeline
- Karma costs prevent excessive character optimization
