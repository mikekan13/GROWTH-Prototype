# Roots Reference Examples

Reference tables of specific Roots with statistical mechanics, providing concrete examples for character background creation. These pre-designed examples show how childhood and upbringing experiences can be mechanically represented.

## Root Mechanics Overview

Roots represent formative childhood experiences and provide:
- **Age Addition:** Years added to character's starting age
- **Karma Cost (KV):** GM karma investment required
- **Wealth Level:** Starting economic resources
- **Skills:** Proficiencies gained from upbringing
- **Attributes:** Modifications to base statistics
- **Social Connections:** Network and reputation effects

## GM Implementation Guidelines

### Age-Wealth-Karma Balance
- Higher KV Roots provide more benefits but cost more karma
- Age addition represents time investment in formative experiences
- Wealth levels affect starting equipment and social opportunities
- Skills reflect realistic development from background experiences

### Customization Notes
- GMs should modify examples to fit campaign setting
- Multiple Roots possible for complex backgrounds (increased age/karma cost)
- Skills and attributes should reflect logical development from experiences
- Social effects create campaign integration opportunities

### Campaign Integration
- Root connections provide ongoing narrative hooks
- Social networks create adventure opportunities
- Background elements should complement party dynamics
- Regional/cultural variations encouraged
