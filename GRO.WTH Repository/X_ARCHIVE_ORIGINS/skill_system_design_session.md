## Key Discovery: Context Over Categories

### The Pattern Recognition
- **Problem:** Rigid skill categories create boundary conflicts and limit creativity
- **Original Intent:** Categories were targeting tags for effects (Nectars, spells, items)
- **Better Solution:** Contextual language in effect descriptions

### Example Evolution
**Old Approach:**
- Heavenly Beauty: +3 to all Social-based Skill Rolls
- Requires rigid "Social Skills" category

**New Approach:**  
- Heavenly Beauty: +3 when using physical attractiveness to influence others
- Works with any freeform skill name in the right context

## Core Skill System Design

### Universal Skill Mechanics
1. **Freeform Skill Names:** Players create any skill name that fits their character
2. **Dice Progression:** Skills progress from d4 → d20 at Level 20 (existing system)
3. **Contextual Effects:** Nectars and other enhancements use natural language triggers
4. **GM Judgment:** GM and player agree on when effects apply to skill usage

### The Multiple Actions Problem
**Initial Concept:** "For every 5 levels in a skill, perform that skill that many times as a single action"
- Levels 1-5: 1 use per action
- Levels 6-10: 2 uses per action  
- Levels 11-15: 3 uses per action
- Levels 16-20: 4 uses per action

**Problem Discovered:** Works for contested skills (combat, social contests) but breaks for non-contested skills (lock picking, research, climbing)

**Clean Solution:** Make multiple actions a Nectar effect rather than universal mechanic

## Final Design Principles

### Core Skills
- **Base Mechanic:** Dice progression only (d4 → d20)
- **Names:** Completely freeform player creation
- **Categories:** None required for players

### Enhanced Effects
- **Combat Speed:** "Flurry Strike" nectar grants multiple attacks per action
- **Social Enhancement:** Effects trigger on contextual usage patterns
- **Specialized Mechanics:** Delivered through nectars, not universal rules

### Effect Writing Template
**Format:** "[Effect Name]: [Bonus] when [contextual trigger in plain English]"

**Examples:**
- Battle Fury: +2 when making weapon attacks
- Scholar's Focus: Reroll failed attempts to recall information  
- Nimble Fingers: Double dice on skills involving manual dexterity

## Development Notes

### What This Solves
- Eliminates skill category conflicts
- Maintains complete creative freedom
- Provides clear mechanical hooks for effects
- Keeps base mechanics simple and universal
- Allows deep customization through optional enhancements

### Backend Considerations
- System may need contextual tagging for digital validation
- Users never see rigid categories
- Effects database uses natural language matching
- GM tools may suggest applicable effects based on described actions

### Future Exploration Needed
- Other mechanics that should be nectars vs. universal rules
- How players discover/acquire contextual enhancement effects
- Integration with Three Pillar Attribute system
- Combat resolution beyond basic attack mechanics

## Session Conclusion

**Core Insight:** GROWTH's strength lies in contextual flexibility rather than rigid systematization. The system should provide mechanical precision through natural language triggers while preserving complete creative freedom for skill naming and usage.

**Next Steps:** Explore nectar creation mechanics and other system components that benefit from this contextual approach.
