
## Core Concept: Post-Capitalist Creative Economy Through Play

GRO.WTH is a Trojan horse for economic revolution, disguised as a TTRPG platform. Players unknowingly build collective ownership through creative play, eventually discovering they're stakeholders in a new economic model.

## The KRMA Economy

### What is KRMA?

- **Influence mass** that gets bound into creations (not spent like currency)
- Represents ownership stake in the GRO.WTH ecosystem
- Tied to real currency (eventually revealed)
- Players own part of the platform through their KRMA accumulation

### KRMA States

1. **Fluid KRMA**: Available for creating new content
2. **Locked KRMA**: Bound into creations (characters, items, worlds)
3. **Soul Package KRMA**: Permanent player ownership after character death

### Subscription Model

- **Only GMs pay subscriptions** (players play free)
- Subscription provides monthly KRMA increase
- Progressive scale:
    - Months 1-12: Building KRMA stake (customer phase)
    - Months 12-24: Break-even (KRMA earnings offset subscription)
    - Months 24+: Net earner (become "employee" of system)

## Attribution Chain System

### The Problem Solved

Addresses the real-world issue of AI training attribution - every creative element is tagged, creators get compensated when their ideas influence outputs.

### How Attribution Works

1. **Creation Registry**
    - Timestamp (immutable)
    - Creator ID (verified account)
    - Content hash (unique fingerprint)
    - Influence tree (parent attributions)
    - Karmic Value (KV) - system-determined power level
    - Usage metrics
2. **Influence Calculation**
    - Creates a DAG (Directed Acyclic Graph) of creative influence
    - When content is generated using multiple sources:
        - A creates "corruption mechanic" → gets 40% attribution
        - B creates "scaling weapon" → gets 30% attribution
        - C combines them → gets 30% attribution for synthesis
        - Future users pay KRMA distributed up the chain
3. **Recursive Attribution**
    - Every derivative work becomes part of the attribution chain
    - Creators earn from direct use AND from derivatives of derivatives
    - Prevents loops through timestamp authority

### Anti-Spam Mechanisms

- **Staking requirement**: Must lock KRMA to publish content
- **Novelty detection**: Generic content has minimal influence weight
- **Godhead curation**: AI suggests relevant content, not keyword matching
- **Saturation scaling**: Higher stakes for oversaturated concepts

## Character Lifecycle & KRMA Flow

### Character Creation

- GM loans KRMA to player for character
- Player operates with borrowed KRMA (not truly owned)

### Character Death - The Graduation System

Death splits KRMA three ways:

1. **Body KRMA** → Returns to GM (loan returned)
2. **Spirit KRMA** → 50% to GM, 50% to soul package
3. **Soul Package** → Player's first TRUE ownership (includes soul content)
4. **Frequency Pool** → Lady Death (system sink, prevents inflation)

**Death is not failure - it's equity graduation!**

### Soul Package Economy

- Persistent IP that can be reincarnated
- Can be used by other players (original owner gets royalties)
- Eventually becomes trainable ML agents
- Permanent investment earning royalties forever

## GM-Player Economic Relationship

### The Incubator Model

1. **GMs** = Venture capitalists investing KRMA
2. **Players** = Creative apprentices building IP
3. **Soul packages** = Equity grants
4. **Graduation** = Players accumulate enough to become GMs

### Incentive Alignment

- GMs want players to succeed (builds value)
- Players want meaningful deaths (graduates KRMA to ownership)
- Death costs GM KRMA (Lady Death's cut) so arbitrary killing is discouraged
- Popular soul packages can "bid" for tables

## Platform Evolution & Media Rights

### Content Generation Pipeline

1. Live play at tables
2. Godhead AI converts sessions to video/media
3. Direct distribution on platform
4. KRMA flows from viewers to creators
5. Attribution chain handles all royalties automatically

### The Creator Economy

- Every table potentially generates franchises
- No middlemen (no studios, no streaming services)
- Players own character IP
- GMs own world/campaign IP
- Merchandise, media rights flow through KRMA attribution
