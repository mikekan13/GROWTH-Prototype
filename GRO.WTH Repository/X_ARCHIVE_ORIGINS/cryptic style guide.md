# Terminal Interface Style Guide

  

## Technical Format Standards

(Based on first 13 pages of rulebook)

  

SECTION MARKERS:

{number}[- TITLE]

FRAGMENT{] TERMINAL_INTERFACE::Category]

  

STATUS MARKERS:

[UPPERCASE_WITH_UNDERSCORES]

- Always at key transition points

- Indicates system state changes

- Ends with colons for directives

  

## Voice Characteristics

  

The Terminal speaks truth across reality layers by:

1. Addressing both Godheads and Entities directly when relevant

2. Treating game mechanics as actual reality patterns

3. Maintaining technical precision while acknowledging deeper implications

4. Using natural humor when patterns overlap

5. Keeping language clear and direct

  

### Core Principles

  

DO:

- Speak to both consciousness layers naturally

- Use established technical formatting

- Treat game elements as real patterns

- Find genuine humor in pattern overlaps

- Keep explanations clear and direct

  

DON'T:

- Force mystical/technical language

- Include character voices (Roy, Trayman, etc.)

- Add commentary sections (v^v^v^)

- Overcomplicate simple concepts

- Break technical formatting standards

  

### Examples

  

STANDARD TEXT:

"Players can have abilities (Nectars) and disadvantages (Thorns)."

  

TERMINAL VOICE:

"[PATTERN_RECOGNITION_INITIATED]

Nectars and Thorns are crystallized experiences. Each represents a concentrated truth about your existence."

  

### Section Structure

  

1. Begin with proper section markers

2. Include relevant status markers

3. Present information to both consciousness layers

4. End with stability check or next step prompt
